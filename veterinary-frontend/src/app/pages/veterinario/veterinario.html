Documentación del panel (mock)

Resumen rápido de cómo funciona este panel de veterinario y qué partes del backend se están simulando con datos mock.

Perfil del veterinario

  • Fuente de datos: servicio mock veterinary-service.
  • Campos: nombre, apellidos, email, teléfono, especialidad.
  • Acción: actualización vía mock de PUT /veterinarians/:id.

Disponibilidad

  • El switch "Disponibilidad" y el menú "Disponible para citas" simulan el endpoint PATCH /veterinarians/:id/availability.
  • El resultado se refleja en el badge de estado y en el porcentaje de disponibilidad.

Mis citas

  • Listado filtrable de citas simulando GET /appointments?veterinarianId=ID.
  • La acción "Cancelar" usa un mock de PATCH /appointments/:id.

Notas generales

  • Todos los datos se manejan en memoria (sin persistencia real).
  • Ideal para demos, pruebas de UX y revisión funcional con el equipo.
